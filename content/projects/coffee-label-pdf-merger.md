*   **專案簡介**：一支給咖啡標籤列印用的桌面小工具。每張標籤是 70×99mm 的單頁 PDF，這支工具負責把數十張不同的標籤挑選、排序、預覽，最後**直接拼成 A4 每頁 9 張（3×3）的列印檔**。用 Python + PySide6(Qt) 開發，可打包成單一 Windows exe，免安裝、免 Python 環境。

## 起點：一個每次必爆的 bug

標籤是用 reportlab 一張一張產出的單頁 PDF。原本的列印流程是：先把很多張標籤合併成一個多頁 PDF，再丟進 Acrobat，靠「多頁併印(N-up)」排成 3×3 印出來。這條路能動，但每次都要在 Acrobat 手動設定邊界、縮放、順序，很容易設錯。

我曾試著讓標籤產生器直接輸出 3×3 的 A4，卻「一直有 bug」。回頭 review 才發現兩層問題：

1. **一個拼字 typo**：`a4` 模式呼叫繪圖函式時把 `image_reader` 打成 `iamge_reader`，而函式簽名沒有這個參數也沒有 `**kwargs`，於是每次跑都直接 `TypeError`，根本產不出檔案。
2. **就算改好拼字，方向也錯**：那段程式是把**同一張**標籤畫 9 次(9 張一樣的)，做不出我要的「9 張**不同**標籤排一頁」。

釐清真正需求後，才決定重寫成一個像樣的工具。

## 核心功能

1. **圖形介面(PySide6 / Qt)**：
    - 選目錄 → 左側列出所有標籤 PDF → 挑選加入「目標清單」。
    - 目標清單**原生拖曳排序**，順序即最終列印順序。
    - 兩塊即時預覽：**左邊**是選取的單張標籤，**右邊**是依目標清單排好的 A4 3×3 版面，增刪或拖曳排序都會自動重算，所見即列印結果。
2. **兩種輸出**：
    - **合併成多頁 PDF** — 相容既有的 Acrobat N-up 習慣。
    - **A4 3×3 拼版(imposition)** — 直接輸出「印出即 3×3」的檔，列印時選「實際大小」即可，**完全不需要 Acrobat 的 N-up**，少一道最容易出錯的手動步驟。
3. **精準對齊**：單張標籤 70×99mm 剛好是 A4(210×297mm)的 1/9，拼版時零縮放、零裁切、精準對齊；尺寸不合的標籤會等比例縮放並置中於格子內，不變形。
4. **打包成單一 exe**：用 PyInstaller 打包成 onefile 的 `PDFMerger.exe`(約 67MB)，分發時只需給一個檔，對方不用裝 Python。

## 幾個工程決策

- **為什麼從 tkinter 換成 Qt**：舊版 tkinter 的介面老舊，而且拖曳排序、PDF 預覽都得手刻。換成 Qt 後，`QListWidget` 的 `InternalMove` 一行就有原生拖曳排序，`QPdfView` 內建原生 PDF 預覽——直接刪掉舊版約 60 行手寫拖曳邏輯和整個自製預覽類別，還少一個渲染相依套件。
- **拼版核心用 pypdf**：以 `merge_transformed_page`(scale + translate)把每張標籤貼到空白 A4 的對應格子，座標以 PDF 左下原點計算、第一列放最上面、讀序為左上→右→下一列。寫成獨立模組，同時提供指令列與程式介面。
- **預覽不落地檔案**：右側的 3×3 預覽如果每次都寫暫存 PDF 再讀，會遇到 Windows 檔案鎖與清理殘留的麻煩。改成把拼版結果產生成 PDF bytes、透過 `QBuffer` **記憶體載入** `QPdfDocument`——沒有暫存檔、沒有鎖、沒有清理問題。
- **即時預覽的節流**：目標清單一有變動就重新拼版有點浪費，加了 400ms 的 debounce，連續拖曳時只在停下來後算一次。
- **打包的坑**：Qt 的 `QtPdf` 是較冷門的模組，PyInstaller 有時不會自動帶它的 DLL，打包後預覽會空白或閃退。所以打包後我用無視窗(offscreen)模式實際啟動 exe 驗證——程式在建立 `QPdfDocument` 前若缺 DLL 就會立刻崩潰，存活即代表預覽模組確實打包成功。
