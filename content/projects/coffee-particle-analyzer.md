*   **專案連結**：[Github simon345wu](https://github.com/simon345wu/particle-analyzer.git)
*   **專案簡介**：咖啡粉粒徑分析系統,包含 **Flutter 跨平台 App**(Android / Windows,已出 release 簽章 APK)與 Python 分析工具鏈。透過數位影像處理自動偵測照片中的每一顆咖啡粉,計算等效圓直徑與粒徑分佈(D10/D50/D90、Dv10/Dv50/Dv90),協助評估磨豆機的研磨均勻度、比較不同刻度的分佈差異。

專案的特色是**雙演算法架構**:除了自行開發的 Otsu + Watershed 管線,還逐行移植了天文學家 Jonathan Gagné 的 [coffeegrindsize](https://github.com/jgagneastro/coffeegrindsize) 演算法(藍色通道背景閾值 + 亮度成本路徑黏粒分離),兩者在 App 內可即時切換,並經過系統性交叉驗證。

## 核心功能

[說明](https://github.com/simon345wu/particle-analyzer#%E6%A0%B8%E5%BF%83%E5%8A%9F%E8%83%BD)

1. **雙演算法可切換(App)**:
    - 內建 Otsu+Watershed 與 CoffeeGrindSize 兩種演算法,一鍵切換。
    - 演算法以 **宣告式參數規格(ParamSpec)** 描述自己的參數,控制面板自動生成對應的滑桿/開關,進階參數收合;新增演算法不需改 UI。
    - 分析在背景 isolate 執行不卡介面,參數調整帶 debounce 即時重算。
2. **自動比例尺與透視校正**:
    - **ArUco 四角標記校正**(10cm 底圖):以標記內緣攤平 + 自動比例尺,對光線與角度最穩健;內建**過曝救援**(自動放寬偵測參數、對比正規化重試)。
    - 15cm 實心方框偵測與手動比例尺模式仍保留備用。
3. **黏連顆粒分離**:
    - Watershed:自適應局部極大值種子 + 分水嶺切割。
    - CoffeeGrindSize:沿像素路徑計算亮度成本,在亮「山脊」處切開黏粒(原作者獨門方法,Dart 移植經逐位元保真驗證)。
4. **以 Dv50 為主指標的統計呈現**:
    - 系統性對比(3 檔研磨度 × 3 重複樣本)證實**體積中位數 Dv50 是對研磨度變化最穩健的指標**:兩演算法皆單調且互相印證(差距 6~10%),App 以主指標卡放大呈現。
    - 完整輸出:數量/體積分佈直方圖(D 線標記)、每顆顆粒明細、結塊佔比品質燈號。
5. **演算法比較平台(psa,Python)**:
    - 模組化比較框架:演算法插拔式接入,共同指標由 label mask 以同一套公式計算(比分割品質,不比量測公式),一鍵產出並排 CSV / 疊圖 / 直方圖報告。
    - 支援共同過濾(尺寸/圓度上限)排除結塊對體積統計的干擾。
6. **工程品質**:
    - 移植保真:Python 移植版與原始 GUI 版逐位元一致;Dart 移植版與 Python 版逐值一致(fixture 測試),真實照片上 Dv50 兩邊完全相同。
    - 合成影像 Ground Truth 驗證、跨 isolate 回歸測試、release 簽章 APK(arm64 48.9MB)。

## 主要發現(來自比較平台的實驗)

- 兩種完全不同的演算法對 Dn50 的判讀每檔差距 <2% — 互為正確性的交叉驗證。
- **拍攝流程比演算法選擇影響更大**:薄層、均勻分散的樣本讓兩演算法收斂;粉量過多黏成片時任何演算法的體積統計都會失真。
- 數量中位數(Dn50)易被細粉偵測差異拉偏,體積中位數(Dv50)才是穩定的研磨度指標。
