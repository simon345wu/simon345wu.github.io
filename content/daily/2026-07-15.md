## 📝 隨手筆記 / 今日心得
- 今天在 RoastWise 加了個小功能:新增烘焙紀錄上傳 `.alog` 檔時,可以順手把同名的曲線圖(jpg/png)一起選進來自動上傳,不用再另外點一次。順便修掉一個 bug——上傳 log 後 Tasting Notes 沒被帶入(原來是解析 log 後把帶入杯測筆記的邏輯擋掉了)。
- 最大的收穫是第一次搞懂並實際用了 **PR(Pull Request)**。以前我都是本機 merge 完直接 push main,Claude 帶我走了一次:推分支 → 開 PR → 在 GitHub 上看整批 diff → 自己按 Merge。差別在於「上線前多一個看清楚再合的關卡」,而且留了紀錄。對單人開發不是必須,但這個專案 push main = 直接部署,多這關比較安心。
- 合併後自動部署成功上線,還學到怎麼用 gcloud + App Hosting API 確認 build/rollout 真的 SUCCEEDED,而不是憑感覺。
- 踩到一個坑:本機的 `main` 其實是以前 Firebase Studio 雲端 IDE 自動 commit 長出來的岔支,跟 GitHub 的主線差了一大截(多 22、少 116 個 commit),一 pull 就整片衝突。搞懂後才知道 **GitHub 的 origin/main 才是真相**,本機那條是殘留。備份後 reset 對齊,乾淨了。
- 心得:代理工具幫忙的同時,自己還是得搞懂底層在幹嘛(PR、分支、部署來源),不然出狀況會慌。今天等於補了一課 git/部署的基本功。
